如何在GitHub上發佈檔案（新手完整教學）

  • 建立帳號與Repository
  • 準備HTML檔案
  • 上傳檔案到GitHub
  • 啟用GitHub Pages
  • 訪問與分享網頁
  • 常見問題

本教學將帶您一步步學習如何建立GitHub帳號、創建儲存庫（Repository）、上傳HTML檔案，以及如何使用GitHub Pages發佈您的網頁，讓其他人能夠透過網址訪問您的網頁內容。我們將以實際案例「wuchishu/story」為範例。

步驟一：建立GitHub帳號與Repository

1.1 註冊GitHub帳號

  1. 前往GitHub官網 https://github.com
  2. 點擊右上角的「Sign up」按鈕
  3. 填寫您的電子郵件、密碼和使用者名稱（例如，我們使用「wuchishu」作為範例）
  4. 完成驗證步驟並確認您的電子郵件
  5. 登入您的GitHub帳號

1.2 創建新的儲存庫（Repository）

  1. 登入後，點擊右上角的「+」圖標，選擇「New repository」
  2. 填寫儲存庫名稱（例如，我們使用「story」作為範例）
  3. 可選：添加儲存庫描述（例如「for a children story」）
  4. 選擇「Public」（公開）選項，讓所有人都能看到您的儲存庫
  5. 勾選「Add a README file」選項，這將自動創建一個README.md文件
  6. 點擊「Create repository」按鈕完成創建

步驟二：準備您的HTML網頁檔案

您需要準備一個HTML檔案作為您的網頁內容。這個檔案可以是您已有的網頁，或者您可以創建一個新的。以下是範例HTML結構：

<!DOCTYPE html>
<html lang="zh-Hant">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>GitHub完整上傳教學 - wuchishu/story</title>
    <style>
        /* 您的CSS樣式 */
    </style>
</head>
<body>
    <header>
        <h1>如何在GitHub上發佈檔案（新手完整教學）</h1>
    </header>
    
    <main>
        <!-- 您的網頁內容 -->
    </main>
    
    <footer>
        © 2025 GitHub教學
    </footer>
</body>
</html>
重要提示： 請確保您的主要HTML檔案名稱為 index.html，這樣GitHub Pages才能正確顯示您的網頁。

步驟三：將檔案上傳到GitHub（完整指令與實例）

您有兩種方式可以上傳檔案：直接在GitHub網頁上傳或使用Git命令行工具。

3.1 方法一：直接在GitHub網頁上傳（最簡單）

  1. 進入您剛創建的儲存庫頁面（例如：https://github.com/wuchishu/story）
  2. 點擊「Add file」按鈕，然後選擇「Upload files」
  3. 拖放您的HTML檔案到上傳區域，或點擊「choose your files」選擇檔案
  4. 在頁面底部填寫提交訊息，例如「新增GitHub教學網頁」
  5. 點擊「Commit changes」按鈕完成上傳

3.2 方法二：使用Git命令行工具

  1. 下載並安裝 Git
  2. 打開終端機或命令提示字元，設定您的Git用戶名和電子郵件： 
    git config --global user.name "wuchishu"
git config --global user.email "您的email@example.com"
  3. 複製（Clone）您的儲存庫到本地： 
    git clone https://github.com/wuchishu/story.git
  4. 假設你在桌面創建一個名為github_tutorial的資料夾，內含一個檔案index.html，將此檔案複製到剛剛下載的資料夾（例如：story）中
  5. 進入儲存庫資料夾： 
    cd story
  6. 添加檔案到Git追蹤： 
    git add .

    這個指令中的「.」（點）代表「目前資料夾中的所有檔案」。git add指令告訴Git開始追蹤story資料夾中的所有新檔案或修改過的檔案。新加入的index.html檔案會被加入到Git的暫存區(staging area)，準備好被提交(commit)。

  7. 提交更改： 
    git commit -m "新增GitHub教學網頁"
  8. 上傳到GitHub： 
    git push origin main

步驟四：啟用GitHub Pages功能

要將您的儲存庫變成可訪問的網頁，您需要啟用GitHub Pages功能：

  1. 在您的儲存庫頁面，點擊頂部選單中的「Settings」（設定）
  2. 在左側選單中，向下滾動找到「Pages」選項（在"Code and automation"部分）
  3. 在「Source」部分，選擇「Deploy from a branch」
  4. 在「Branch」下拉選單中，選擇「main」分支和「/ (root)」資料夾
  5. 點擊「Save」按鈕保存設定
  6. 等待幾分鐘，GitHub完成部署
注意： GitHub Pages需要幾分鐘時間來部署您的網站。請耐心等待。

步驟五：訪問與分享您的網頁

一旦GitHub Pages功能啟用並完成部署，您將獲得一個可訪問的網址：

https://[您的GitHub用戶名].github.io/[儲存庫名稱]/

例如，根據我們的範例，網址將是：

https://wuchishu.github.io/story/

現在，任何人都可以通過這個網址訪問您的網頁了！

常見問題處理

我的檔案名稱不是index.html，網頁無法正常顯示

解決方法：

  1. 在儲存庫中點擊您的HTML檔案
  2. 點擊編輯按鈕（鉛筆圖標）
  3. 在編輯界面的檔案名稱欄位中，將檔案名稱改為「index.html」
  4. 在頁面底部填寫一個提交訊息（例如「重新命名為 index.html」）
  5. 點擊「Commit changes」按鈕

上傳後無法看到我的網頁

可能原因和解決方法：

  • GitHub Pages可能需要幾分鐘時間來部署您的網頁
  • 確認您已經正確啟用GitHub Pages功能
  • 確認您的HTML檔案命名為「index.html」
  • 檢查網址是否正確（https://[用戶名].github.io/[儲存庫名]/）

如何添加其他檔案（如圖片、CSS或JavaScript）

您可以按照相同的方式上傳其他檔案到您的儲存庫，然後在HTML檔案中引用它們：

  • 圖片：<img src="圖片檔案名稱.jpg" alt="描述">
  • CSS：<link rel="stylesheet" href="樣式檔案名稱.css">
  • JavaScript：<script src="腳本檔案名稱.js"></script>
