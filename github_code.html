簡介

本教學將介紹如何從本地電腦（以您的檔案路徑為例）將HTML檔案上傳到GitHub，並部署為可訪問的網頁。我們將使用Git命令行工具來完成這個過程。

您的檔案資訊：
檔案路徑：D:\114\code\github_test\story\github_deployment.html
檔案大小：13 KB
修改日期：2025/4/1 上午 06:05

前置準備

步驟 1：安裝 Git

如果您尚未安裝Git，請從Git官網下載並安裝。

步驟 2：配置 Git 用戶信息

開啟命令提示字元（CMD）或PowerShell，執行以下命令設定您的Git用戶名和電子郵件：

git config --global user.name "wuchishu"
git config --global user.email "您的email@example.com"

步驟 3：確認您已有GitHub帳號和儲存庫

確保您已經有一個GitHub帳號（wuchishu）和一個儲存庫（story）。如果沒有，請先創建。

從GitHub下載儲存庫到本地

步驟 4：選擇或創建本地資料夾

根據您的路徑，您已經在以下位置創建了資料夾：

D:\114\code\github_test\story

如果您尚未創建這個資料夾結構，您需要先創建它。

步驟 5：複製（Clone）GitHub儲存庫

使用CMD或PowerShell，導航到您想要放置儲存庫的父資料夾，然後克隆儲存庫：

cd D:\114\code\github_test
git clone https://github.com/wuchishu/story.git

這將會從GitHub下載story儲存庫到您的本地資料夾。如果資料夾已經存在，且已經有.git資料夾（表示已經是一個Git儲存庫），則不需要再次克隆。

添加HTML檔案到儲存庫

步驟 6：確認您的HTML檔案

您已經創建了兩個HTML檔案：

  • github-tutorial.html (5 KB)
  • github_deployment.html (13 KB)

這些檔案位於 D:\114\code\github_test\story 資料夾中。

步驟 7：如果需要，重命名其中一個檔案為 index.html

要讓GitHub Pages正確顯示您的網頁，主頁文件應該命名為 index.html。您可以選擇將其中一個檔案重命名，或者新建一個index.html檔案。

重命名檔案的命令：

rename github_deployment.html index.html

或者，您也可以保留現有檔案名稱，但訪問時需要在URL後面指定檔案名。

提交更改並上傳到GitHub

步驟 8：打開命令提示字元並導航到儲存庫資料夾

cd D:\114\code\github_test\story

步驟 9：將檔案添加到Git追蹤

執行以下命令將您的HTML檔案添加到Git的暫存區：

git add github-tutorial.html github_deployment.html

或者添加所有檔案：

git add .

紅色圓圈圖標（如您截圖中所示）表示這些檔案尚未被Git追蹤。執行上述命令後，這些圖標會變為綠色，表示檔案已被添加到暫存區。

步驟 10：提交更改

執行以下命令提交您的更改：

git commit -m "新增GitHub教學和部署指南HTML檔案"

這個命令將創建一個新的提交，包含您添加的檔案，並附上一條描述訊息。

步驟 11：上傳到GitHub

執行以下命令將您的更改推送到GitHub：

git push origin main

系統可能會要求您輸入GitHub帳號和密碼或個人訪問令牌。

部署為GitHub Pages網頁

步驟 12：前往GitHub網站啟用Pages功能

  1. 打開瀏覽器，前往您的GitHub儲存庫頁面：https://github.com/wuchishu/story
  2. 點擊「Settings」（設定）
  3. 在左側選單中，向下滾動找到「Pages」選項
  4. 在「Source」部分，選擇「Deploy from a branch」
  5. 在「Branch」下拉選單中，選擇「main」分支和「/ (root)」資料夾
  6. 點擊「Save」按鈕保存設定

步驟 13：等待部署完成

GitHub需要幾分鐘時間來部署您的網站。部署完成後，您將在同一頁面看到一個綠色橫幅，顯示您的網站URL。

步驟 14：訪問您的網頁

您的網頁現在可以通過以下URL訪問：

https://wuchishu.github.io/story/

如果您沒有將檔案重命名為index.html，則需要在URL後面加上檔案名：

https://wuchishu.github.io/story/github_deployment.html
https://wuchishu.github.io/story/github-tutorial.html

常見問題與解決方法

問題：push時要求輸入帳號密碼

解決方法： GitHub不再支持使用密碼進行Git操作。您需要創建一個個人訪問令牌（Personal Access Token）：

  1. 前往GitHub的Settings > Developer settings > Personal access tokens
  2. 點擊"Generate new token"
  3. 設定token的有效期和權限（至少需要repo權限）
  4. 生成token後，在push時使用此token作為密碼

問題：無法看到我的網頁或顯示404錯誤

解決方法：

  • 確保GitHub Pages功能已正確啟用
  • 確保您有一個名為index.html的檔案，或者在URL中指定正確的檔案名
  • GitHub Pages部署可能需要幾分鐘時間，請耐心等待
  • 檢查您的儲存庫名稱和用戶名是否在URL中正確拼寫

問題：檔案更新後網頁沒有更新

解決方法：

  • 確保您已經提交(commit)並推送(push)了更改
  • 清除瀏覽器緩存，或使用無痕模式訪問
  • 等待幾分鐘，GitHub Pages可能需要時間來更新
